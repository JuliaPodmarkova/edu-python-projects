# Модуль 11: Интроспекция и рефлексия в Python

В этой папке собраны программы, демонстрирующие возможности интроспекции (рефлексии) в Python: получение информации об объектах, типах, атрибутах, функциях, модулях и классах во время выполнения. Используются встроенные функции (`type`, `isinstance`, `hasattr`, `getattr`, `callable`, `dir`, `pprint`) и модули `inspect` и `sys` для более глубокого анализа.

## 📂 Содержание

| Файл | Что делает | Демонстрирует |
|------|------------|----------------|
| `Lection4.py` | Базовые операции интроспекции: вывод имён объектов (`__name__`), проверка типов (`type`), просмотр атрибутов модуля `requests` и пользовательского объекта с помощью `pprint(dir())`. | `__name__`, `type`, `dir()`, `pprint`, анализ стороннего модуля (`requests`) |
| `Lection5.py` | Расширенная интроспекция: проверка наличия атрибута (`hasattr`), получение значения атрибута (`getattr`), проверка вызываемости (`callable`), проверка принадлежности классу (`isinstance`), анализ модуля `requests` через `inspect` (класс, модуль, функция, абстрактность, встроенность), получение модуля функции (`inspect.getmodule`). | `hasattr`, `getattr`, `callable`, `isinstance`, `inspect.isclass`, `inspect.ismodule`, `inspect.isfunction`, `getmodule` |
| `Lection6.py` | Определение операционной системы через `sys.platform`. | `sys.platform` |
| `full_code.txt` | Пустой файл (не используется). | – |

## 🚀 Как запустить любой скрипт

1. Откройте терминал в папке `ModulsOfTema1/Module11` (или там, где лежат эти файлы).
2. Выполните:
   ```bash
   python Lection4.py
   python Lection5.py
   python Lection6.py
   ```

## 📦 Зависимости

- **Стандартная библиотека Python**: используется везде.
- **Библиотека `requests`**: используется в `Lection4.py` и `Lection5.py` для демонстрации интроспекции стороннего модуля. Перед запуском этих скриптов установите её:
  ```bash
  pip install requests
  ```
  Если `requests` не установлен, вы получите `ModuleNotFoundError`.

Все остальные модули (`pprint`, `inspect`, `sys`) входят в стандартную поставку Python.

## 🧠 Ключевые концепции

- **Интроспекция** – способность программы узнавать информацию об объектах во время выполнения.
- **Атрибуты объектов** – `__name__`, `__dict__`, `dir()`.
- **Проверка типов** – `type()`, `isinstance()`.
- **Проверка наличия и получения атрибутов** – `hasattr()`, `getattr()`.
- **Проверка вызываемости** – `callable()`.
- **Модуль `inspect`** – предоставляет функции для анализа живых объектов (классы, функции, модули, код).
- **Модуль `sys`** – доступ к системным параметрам, например, `sys.platform`.
