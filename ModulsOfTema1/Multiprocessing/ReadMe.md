# Модуль 12: Многопроцессорность (`multiprocessing`) и параллелизм в Python

В этой папке собраны программы, демонстрирующие работу с многопроцессорностью (`multiprocessing`), многопоточностью (`threading`), межпроцессным взаимодействием (`Pipe`, `Queue`), а также практические примеры обработки изображений и работы с API Genius. Материалы показывают разницу между потоками и процессами, синхронизацию, создание пулов процессов и обмен данными.

## 📂 Содержание

| Файл | Что делает | Ключевые концепции |
|------|------------|---------------------|
| `Lection1.py` | Сравнение потоков и процессов: два рабочих инкрементируют глобальный счётчик. В коде закомментирована версия с потоками, активна – с процессами. | `multiprocessing.Process`, глобальная переменная, разделение памяти (в процессах не共享) |
| `Files/HomeWork.py` | Сравнение линейного чтения четырёх больших файлов и многопроцессорного с помощью `Pool.map()`. Замер времени выполнения. | `multiprocessing.Pool`, `map`, замер времени (`datetime`), работа с файлами |
| `practic.py` | Обработка изображений: изменение размера и преобразование в ч/б. Два процесса обмениваются данными через `multiprocessing.Queue`. | `multiprocessing.Process`, `multiprocessing.Queue`, обработка исключений `Empty`, `timeout` |
| `practic(pipe).py` | Аналогичная обработка изображений, но используется `multiprocessing.Pipe` и `Event` для синхронизации. | `Pipe`, `Event`, двусторонняя связь между процессами |
| `practic1.py` | Имитация производства булок и приготовления котлет (бургеров). Используются **потоки** (`Thread`) и очередь `queue.Queue`. Генератор булок кладёт в очередь, потребитель забирает. | многопоточность, `queue.Queue`, `Thread`, `qsize()`, `timeout` в `get()` |
| `practic2-1.py` | Пример работы с API Genius: получение случайного жанра, поиск песни, вывод ссылки на музыкальный плеер. **Без параллелизма**. | HTTP-запросы (`requests`), работа с JSON |
| `practic2-2.py` | Многопоточный сбор песен: несколько потоков `GetGenre` генерируют жанры и кладут в очередь, несколько потоков `Genius` забирают жанры, запрашивают API и сохраняют результаты в общий список `all_songs`. | `Thread`, `queue.Queue`, `Event`, синхронизация завершения |
| `practic2-2(counter).py` | Улучшенная версия: потоки `Genius` завершаются после сбора заданного количества песен (`counter`). Используется `stop_event` для остановки производителей. | условная остановка потоков, счётчик, общий список результатов |
| `Files/file 1.txt` … `file 4.txt` | Тестовые файлы с большим количеством строк (числа) для замера скорости чтения в `HomeWork.py`. | тестовые данные |

> **Примечание**: файлы `practic2-2.py` и `practic2-2(counter).py` используют **токен доступа** к API Genius. Предоставленные токены могут быть недействительными. Для работы нужно получить свой токен на [Genius API](https://docs.genius.com/).

## 🚀 Как запустить любой скрипт

1. Откройте терминал в папке, содержащей скрипты.
2. Установите необходимые зависимости (см. ниже).
3. Выполните, например:
   ```bash
   python Lection1.py
   python Files/HomeWork.py
   python practic.py
   ```

Обратите внимание: для `practic.py` и `practic(pipe).py` требуется наличие папки `images` с изображениями `2.jpg` … `10.jpg` (или измените пути в коде).

## 📦 Зависимости

- **Стандартная библиотека Python** – используется везде (`multiprocessing`, `threading`, `queue`, `datetime`, `random`, `time`, `pprint`).
- **Pillow (PIL)** – для обработки изображений в `practic.py` и `practic(pipe).py`. Установка:
  ```bash
  pip install Pillow
  ```
- **Requests** – для работы с API Genius в `practic2-1.py`, `practic2-2.py`, `practic2-2(counter).py`. Установка:
  ```bash
  pip install requests
  ```

Для запуска `HomeWork.py` убедитесь, что файлы `file 1.txt` … `file 4.txt` лежат в папке `Files` относительно скрипта, и они содержат достаточное количество строк (например, числа от 1 до 9 999 999).

## 🧠 Ключевые концепции

- **Многопроцессорность** – создание процессов через `multiprocessing.Process`, использование пула (`Pool`), обмен данными через `Queue` и `Pipe`.
- **Синхронизация процессов** – `Event`, блокировки.
- **Сравнение потоков и процессов** – потоки разделяют память (проблема гонок), процессы – изолированы.
- **Очереди** – потокобезопасные (`queue.Queue`) и процесс-безопасные (`multiprocessing.Queue`).
- **Работа с API** – асинхронный (через потоки) сбор данных.
- **Обработка изображений** – изменение размера, преобразование в оттенки серого (градации серого).
