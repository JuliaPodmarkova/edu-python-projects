# Модуль 10: Многопоточность в Python

В этой папке собраны программы, демонстрирующие работу с потоками (`threading`) в Python: создание и запуск потоков, синхронизация (блокировки, `Lock`), состояние гонки (`race condition`), использование очередей (`Queue`), событий (`Event`), обработка исключений в потоках, влияние GIL на производительность, а также практические задачи по имитации работы банка и кафе.

## 📂 Содержание

| Файл | Что делает | Демонстрирует |
|------|------------|----------------|
| `Lection1.py` | Базовые операции: получение информации о текущем потоке, список активных потоков, запуск потока с демонизацией (`daemon=True`), ожидание завершения (`join()`). | `threading.current_thread()`, `threading.enumerate()`, `Thread.start()`, `Thread.join()`, `is_alive()`, `daemon` |
| `Lection2.py` | Создание собственного класса потока, наследующего `threading.Thread`. Переопределение метода `run()`. | наследование от `Thread`, `__init__`, переопределение `run()` |
| `Lection3.py` | Демонстрация состояния гонки (`race condition`) на примере увеличения глобальной переменной двумя потоками. Исправление с помощью `Lock`. Также пример взаимной блокировки (`deadlock`) с двумя блокировками. Перечисление атомарных операций в Python. | `race condition`, `Lock`, `acquire()`/`release()`, `finally` для гарантированного освобождения, deadlock |
| `Lection4/` | **Исследование влияния GIL** на многопоточные вычисления: четыре больших JSON-файла с массивами чисел. Обычный последовательный подсчёт суммы и многопоточный (4 потока) с замером времени. | GIL, многопоточность для I/O vs CPU, `Thread`, замер времени (`datetime`) |
| `Lection5_part1.py` | Обработка исключений в потоках: исключение, возникшее в потоке, не выводится в консоль, если не перехвачено. | исключения в потоках, перехват внутри `try/except` |
| `Lection5_part2.py` | Установка глобального перехватчика исключений для потоков через `threading.excepthook`. | `threading.excepthook`, обработка непойманных исключений |

| `Lection7.py` | Синхронизация потоков с помощью `Event`. Два рабочих потока: второй сигнализирует первому через `event.set()`. | `threading.Event`, `wait()`, `set()`, `is_set()` |
| `Module10_HW1.py` | **Домашнее задание 1**: имитация работы банка. Поток `deposit` пополняет счёт, поток `take` снимает средства. Используется `Lock` для предотвращения одновременного доступа. | практика `Lock`, синхронизация доступа к общему ресурсу (`balance`) |
| `Module10_HW2.py` | **Домашнее задание 2**: имитация кафе. Гости (`Guest` – потоки) садятся за свободные столы (`Table`), иначе становятся в очередь (`Queue`). | `Thread`, `Queue`, взаимодействие потоков, атрибуты объектов |
| `Lection4/file1.json` … `file3.json` | Большие JSON-файлы с массивами случайных чисел (по 10000 элементов), используются для экспериментов с GIL и замера времени. | тестовые данные для многопоточных вычислений |

## 🚀 Как запустить любой скрипт

1. Откройте терминал в папке `ModulsOfTema1/Module10`.
2. Выполните (примеры):
   ```bash
   python Lection1.py
   python Lection2.py
   python Lection3.py
   python Lection4/Lection4.py
   python Lection4/Lection4_gil.py
   python Lection4/threads_count.py
   python "Lection5/Lection5_part1.py"
   python "Lection5/Lection5_part2.py"
   python Lection6.py
   python Lection7.py
   python Module10_HW1.py
   python Module10_HW2.py
   ```

> **Примечание**: Скрипты `Lection4/Lection4_gil.py` и `threads_count.py` используют файлы `file1.json`, `file2.json`, `file3.json`, `file4.json`, которые уже лежат в папке `Lection4`. Убедитесь, что они доступны по указанным путям.

## 📦 Зависимости

Все скрипты используют только стандартную библиотеку Python. Дополнительных установок не требуется. Для работы с JSON используется модуль `json`, для HTTP-запросов – `requests` (но в предоставленных файлах он импортирован, но не используется; для запуска `Module10_HW1.py` он не нужен).

## 🧠 Ключевые концепции

- **Потоки**: создание, запуск, ожидание, демонизация.
- **Синхронизация**: `Lock`, `Event`, `Queue` (потокобезопасная очередь).
- **Race condition и deadlock**: причины и способы предотвращения.
- **GIL (Global Interpreter Lock)**: влияние на производительность CPU-связанных задач.
- **Обработка исключений** в потоках через `threading.excepthook`.
- **Практические задачи**: банк (баланс и блокировки), кафе (столы и очередь).
