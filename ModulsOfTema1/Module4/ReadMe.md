# Модуль 4: модули, пакеты, пространства имён, алгоритмы сортировки

В этой папке собраны программы, демонстрирующие работу с модулями (импорт, переименование, `__name__`), пакетами (`__init__.py`), областями видимости (`global`, `nonlocal`), а также реализацию трёх базовых алгоритмов сортировки.

## 📂 Содержание

| Файл / Папка | Что делает | Демонстрирует |
|--------------|------------|----------------|
| `Lections4Mod.py` | Импортирует функцию `sey_hi` из `secondModuleToLections.py` (через псевдоним `sh`), выводит пути поиска модулей (`sys.path`) | именованный импорт, псевдонимы (`as`), `sys.path` |
| `secondModuleToLections.py` | Вспомогательный модуль: функция `sey_hi()` печатает приветствие и случайное число, функция `some_func()` возвращает строку, блок `if __name__ == '__main__'` | создание модуля, `__name__`, функция `dis` для дизассемблирования |
| `LectionsNamespaceAndScopes.py` | Примеры областей видимости: `global`, `nonlocal` внутри вложенной функции, изменение внешних переменных | `global`, `nonlocal`, вложенные функции |
| `module_4_2.py` | Демонстрация вложенной функции `inner_function()`, которая видна только внутри `test_function()` (вызов вне её вызывает ошибку) | замыкания, область видимости вложенных функций |
| `Module_4/` (пакет) | Пакет с модулем `alg.py`, содержащим три алгоритма сортировки: пузырьковая (`bubbleSort`), выбором (`selectionSort`), вставкой (`insertionSort`) | создание пакета, `__init__.py`, импорт из пакета |
| `ins.py` | Импортирует все функции из `Module_4.alg`, запрашивает у пользователя три списка чисел, сортирует каждый своим методом и выводит результаты | импорт всего модуля, вызов функций сортировки, работа с `input()` |
| `module_4_1/` (пакет) | Пакет с двумя модулями: `fake_math.py` (возвращает `'Ошибка'` при делении на ноль) и `true_math.py` (возвращает `inf` из `math`). | создание пакета, различные обработки деления на ноль |
| `module_4_1/module_4_1.py` | Импортирует `divide` из обоих модулей с псевдонимами `fake_divide` и `true_divide`, демонстрирует их работу | импорт из пакета, алиасы, сравнение поведения |

## 🚀 Как запустить любой скрипт

1. Откройте терминал в папке `ModulsOfTema1/Module4`.
2. Для запуска конкретного файла выполните, например:
   ```bash
   python Lections4Mod.py
   ```
   ```bash
   python ins.py
   ```
   ```bash
   python module_4_2.py
   ```
   ```bash
   python -m module_4_1.module_4_1   # запуск модуля внутри пакета
   ```

> **Обратите внимание**: Файлы `Lections4Mod.py` и `ins.py` содержат импорты из соседних папок (`Module_4`, `secondModuleToLections`), поэтому важно не перемещать их по отдельности, а сохранять предложенную иерархию.

## 📦 Зависимости

- Все скрипты используют только стандартную библиотеку Python.
- Для работы `ins.py` необходим пакет `Module_4` с файлом `__init__.py` (он уже есть).
- Для `module_4_1.py` необходим пакет `module_4_1` с обоими модулями.
