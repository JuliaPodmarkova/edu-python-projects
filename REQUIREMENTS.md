# Технические требования к проектам

## Структура и оформление
- Каждая лекция и домашняя работа лежит в своей папке: `ModulsOfTema1/ModuleN`.
- Внутри папки — код на Python (`*.py`) и отдельный `README.md` с описанием задачи.
- Запрещены:
  - неинформативные имена файлов (`temp.py`, `new.py`);
  - закомментированный «мусор» и неиспользуемые импорты;
  - файлы среды разработки (`.idea`, `.vscode`, `__pycache__`, `.DS_Store`).

## Качество кода
- Код должен выполняться без ошибок на Python 3.10+.
- Следовать PEP8 (проверяется через `flake8` или `pylint`).
- Функции и классы должны иметь docstring (хотя бы кратко).
- Переменные называются на английском, осмысленно.

## Документация
- В корневом `README.md` — общее описание и быстрый старт.
- Внутри каждого модуля `README.md` — условие задачи и пример запуска.

## Git и GitHub
- Коммиты: атомарные, с осмысленными сообщениями на русском или английском.
- Ветка `main` — только стабильные, проверенные версии.
- Файлы, которые не должны попадать в репозиторий, перечислены в `.gitignore`.
